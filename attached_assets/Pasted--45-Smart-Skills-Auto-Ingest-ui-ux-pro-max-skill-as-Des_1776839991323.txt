#45 - Smart Skills: Auto-Ingest ui-ux-pro-max-skill as Design Intelligence + UX Lane Overlays

What & Why

FLOATR should not require a human to manually import high-value skill repos one by one when they are strategically important to the product.

This task adds a curated first-party ingestion pipeline for github.com/nextlevelbuilder/ui-ux-pro-max-skill, converting it into FLOATR-native assets automatically at startup or via an internal seed/import job.

This repo is a strong candidate for first-party ingestion because it contains a large, structured UI/UX intelligence set — including 67 UI styles, 161 color palettes, 57 font pairings, 25 chart types, 15 tech stacks, 99 UX guidelines, and 161 reasoning rules — and it is organized with multiple source surfaces including .claude/skills, CLAUDE.md, skill.json, docs, cli, and source code.

Done looks like
Automatic curated import exists

On API startup or via a one-shot internal seeding job, FLOATR automatically:

fetches nextlevelbuilder/ui-ux-pro-max-skill
records source metadata and pinned commit SHA
parses supported source artifacts
stores normalized assets without requiring manual user import through the dashboard

This repo is handled as a curated built-in source, not a normal ad hoc community import.

Source decomposition is implemented

FLOATR splits the upstream repo into 3 asset classes:

1. Prompt / manifest assets

Examples:

reasoning guidance
design-system generation guidance
review checklists
anti-pattern instructions
stack-specific frontend guidance

These become FLOATR skill manifests.

2. Structured design datasets

Examples:

style taxonomy
palette mappings
typography pairings
chart-type guidance
product-type reasoning rules
UX guidelines
anti-pattern tables

These are stored in structured form and indexed for retrieval.

3. Heavy runtime generators

Examples:

logo / icon / banner / CIP / slides / social creative workflows
repo CLI-driven generators
any nontrivial image/media flows

These are not added to the default runtime. They are marked as optional heavy creative jobs.

FLOATR-native derived skills are created

At minimum, FLOATR auto-creates and seeds these derived skills:

design-intelligence-core
ui-ux-reasoning
design-system-scaffold
frontend-design-review
dashboard-viz-guidance
design-handoff-discipline

These are first-party FLOATR skills derived from the upstream repo, not raw copies of its upstream install model.

Lane integration works automatically

The new skills are auto-ranked for:

ux lanes
frontend / general lanes where frontend is detected
review lanes when UI-heavy files are in scope

They are not added to backend/debug lanes by default, except for limited shared design-contract awareness when relevant.

Shared repo design intelligence exists

A shared repo-level design knowledge layer is available for:

product-type → style guidance
style → anti-pattern guidance
chart-type recommendations
design-system heuristics
frontend stack-specific conventions

This knowledge is retrievable through FLOATR’s existing repo/memory retrieval model instead of being pasted as one giant prompt.

No raw upstream CLI install by default

The upstream repo includes a CLI and generated platform-specific install flow, but FLOATR does not install that CLI globally into sessions by default. It is ingested and normalized centrally instead.
