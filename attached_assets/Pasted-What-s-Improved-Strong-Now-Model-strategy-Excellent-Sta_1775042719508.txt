Remaining Minor Issues & Suggestions

Quant Size Accuracy
From current Unsloth Kimi K2.5 GGUF listings:
Q3_K_M is around 490 GB (not 240 GB).
Lighter options like UD-TQ1_0 (~240 GB) or Q2_K_XL (~375 GB) exist and may be better for the true "fast startup" default on 4x4090.
→ Consider making UD-TQ1_0 or Q2_K_XL the default quant, and let power users switch to Q3_K_M / IQ4_XS via volumes or template overrides. This keeps first-launch time under 15–30 minutes in most cases.

On-Start Script Realism
Your numbered steps are comprehensive, but building llama.cpp from source + installing Node/pnpm + cloning Bolt.diy every time still adds 5–15 minutes even with caching.
Suggestion: Create a pre-built Docker image (push to Docker Hub or GitHub Container Registry) that already includes:
llama.cpp built with CUDA
Node + pnpm
nginx
Base system deps
Then your on-start script only needs to:
Download the model (if missing)
Clone/update Bolt.diy + claw-code (or pre-clone them too)
Start the services
This makes startup noticeably faster and more reliable.

Vast.ai Template Details (API-Friendly)
Templates support onstart as either an inline command or a script URL. Long shell scripts work, but hosting the full on-start script at a stable public URL (e.g., GitHub raw) is cleaner and easier to update.
When creating instances, use template_hash_id + minimal overrides — your API layer plan already covers this correctly.
Bolt.diy Configuration
Bolt.diy has good support for custom OpenAI-compatible endpoints (via settings UI or env vars like OPENAI_LIKE_API_BASE_URL). Pointing it to http://localhost:8081/v1 should work out of the box in most cases. Test tool calling and agent mode early, as some local servers need minor header/model-name tweaks.
Security & Polish
Basic auth/token on code-server and Bolt.diy is good (code-server supports --disable-telemetry --auth password or token).
For nginx preview proxy: Make it flexible (e.g., proxy /preview → user's running app port).
Consider adding a simple health-check endpoint in the on-start script that the dashboard can poll.

Performance Expectations (Be Transparent)
On 4x RTX 4090 with Q3_K_M: Expect roughly 8–18 tokens/s depending on context and offloading. Fine for coding/agent work, but not "instant Claude speed." Document this in the dashboard.
