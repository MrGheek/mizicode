Build a production-ready MVP called TourForge.

TourForge is a browser-based capture-to-hosted-3D-tour platform.

It is NOT a real estate listings website.

The goal is to let an agent/client open a mobile web capture assistant, record guided property footage, upload it, create a processing job, and then let an internal operator/GPU worker produce a hosted interactive 3D Gaussian Splat tour that can be shared as a link or embedded in any third-party website as an iframe.

Use:
- Next.js App Router
- TypeScript
- Tailwind CSS
- Supabase database
- Supabase Storage
- PWA/mobile-first design
- Dark premium UI
- No authentication for MVP, but add TODO comments for auth

Core MVP modules:

1. Mobile Browser Capture Assistant
2. Internal Processing Dashboard
3. GPU Worker Integration Placeholder
4. Hosted Tour Viewer
5. Iframe Embed Output

Do NOT build:
- Public property marketplace
- Agent CRM
- Buyer lead capture
- Real-estate listing search
- Payments
- Native Android app

GPU_PIPELINE_PLAN.md

Create this file and explain future automation:

Pipeline:
1. Download raw footage from Supabase Storage
2. Extract frames with FFmpeg
3. Run COLMAP for camera poses
4. Run Gaussian Splat reconstruction using video-3d-reconstruction-gsplat or gsplat
5. Export PLY/SPLAT
6. Optimize/compress using PlayCanvas SplatTransform or SuperSplat tooling
7. Upload final assets to Supabase Storage
8. Call /api/gpu-callback with optimized_splat_url and status published

Include example future worker command:

python worker.py \
  --job-id JOB_ID \
  --video-url RAW_FOOTAGE_URL \
  --callback-url APP_URL/api/gpu-callback

Also include recommended future GPU providers:
- RunPod
- Vast.ai
- Modal
- Lambda Labs

Important:
- Make the app functional without real GPU processing.
- The MVP should support manual final splat upload and external viewer URL insertion.
- The capture assistant must work in mobile browser.
- The output must be a hosted interactive tour URL and iframe embed page.
- Focus on the operational workflow:
  guided capture → upload → processing job → hosted tour → iframe.
